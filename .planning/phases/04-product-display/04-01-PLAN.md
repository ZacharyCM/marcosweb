Rewrite proxy.ts from the current one-liner re-export into a full proxy function that enforces BOTH the age-gate cookie check AND the existing auth gate. Keep the same `config.matcher`.

Logic order (top to bottom, first match wins):
1. If the path is `/age-gate`, `/privacy`, or starts with `/_next`, `/api`, `/favicon.ico`, `/studio` — call `NextResponse.next()` immediately (no checks). Age-gate and privacy must be reachable without a cookie.
2. If `request.cookies.has('age-verified')` is FALSE — redirect to `/age-gate`. This runs BEFORE the auth check so unverified visitors never see the login form or any app content.
3. If age is verified, delegate to the Auth.js auth function for the remaining auth gate logic.

The Auth.js `auth` function cannot be used as a simple re-export anymore. Import it as the callback form:

```typescript
import { auth } from "@/auth"
import { NextResponse } from "next/server"
import type { NextRequest } from "next/server"

const PUBLIC_PATHS = ["/age-gate", "/privacy"]
const ASSET_PREFIXES = ["/_next", "/api", "/favicon.ico", "/studio"]

export const proxy = auth(function proxyHandler(req) {
  const { pathname } = req.nextUrl

  // Step 1: Always allow public/asset paths through
  if (
    PUBLIC_PATHS.some((p) => pathname.startsWith(p)) ||
    ASSET_PREFIXES.some((p) => pathname.startsWith(p))
  ) {
    return NextResponse.next()
  }

  // Step 2: Age gate — redirect if cookie missing
  if (!req.cookies.has("age-verified")) {
    return NextResponse.redirect(new URL("/age-gate", req.url))
  }

  // Step 3: Auth gate — auth() callback provides req.auth
  // Auth.js handles unauthenticated → /login and non-approved → /login?code=pending/denied
  // If no redirect needed, auth() returns undefined → Next.js continues
})

export const config = {
  matcher: ["/((?!api|_next/static|_next/image|favicon.ico|studio).*)"],
}
```

NOTE: The `auth()` callback form (wrapping proxyHandler) is how Auth.js v5 supports custom proxy logic while still running the auth check. `req.auth` is populated by Auth.js. The existing auth.config.ts callbacks handle the redirect logic — they fire inside auth() when auth state is evaluated.

Verify auth.ts exports `auth` correctly (it does — confirmed in codebase read).

**src/app/actions/age-gate.ts** — Server Action that sets the age-verified cookie and redirects to /login:
```typescript
"use server"
import { cookies } from "next/headers"
import { redirect } from "next/navigation"

export async function confirmAge(): Promise<void> {
  const cookieStore = await cookies()
  cookieStore.set("age-verified", "1", {
    maxAge: 60 * 60 * 24 * 365, // 365 days
    path: "/",
    sameSite: "lax",
    httpOnly: true,
    secure: process.env.NODE_ENV === "production",
  })
  redirect("/login")
}
```
Note: redirect() throws NEXT_REDIRECT — do NOT wrap in try/catch. No return value needed.
