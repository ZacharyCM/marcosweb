  <action>
Create `src/sanity/lib/queries.ts`. Import `defineQuery` from `groq` (NOT from `next-sanity` — use the `groq` package for TypeGen compatibility). Write every query as a named exported constant using `defineQuery(...)`.

Critical rules per research:
- Every query MUST use `defineQuery(...)` assigned to a named export — inline query strings in `client.fetch()` are invisible to TypeGen.
- Every media item projection MUST use `asset->url` (dereference operator) — never return raw `_ref` objects.
- Use `_type == "image"` and `_type == "file"` to distinguish media types in projections.

```typescript
import { defineQuery } from 'groq'

// All products — full detail including resolved media URLs
export const ALL_PRODUCTS_QUERY = defineQuery(`
  *[_type == "product"] | order(name asc) {
    _id,
    _type,
    name,
    strainType,
    price,
    thcPercent,
    cbdPercent,
    description,
    effects,
    "media": media[] {
      _type,
      _key,
      alt,
      caption,
      "url": asset->url,
      "mimeType": asset->mimeType
    }
  }
`)

// Single product by _id — full detail
export const PRODUCT_BY_ID_QUERY = defineQuery(`
  *[_type == "product" && _id == $id][0] {
    _id,
    _type,
    name,
    strainType,
    price,
    thcPercent,
    cbdPercent,
    description,
    effects,
    "media": media[] {
      _type,
      _key,
      alt,
      caption,
      "url": asset->url,
      "mimeType": asset->mimeType
    }
  }
`)

// Products by strain type — card-level fields only (used for Phase 4 carousels)
export const PRODUCTS_BY_STRAIN_QUERY = defineQuery(`
  *[_type == "product" && strainType == $strainType] | order(name asc) {
    _id,
    name,
    strainType,
    price,
    thcPercent,
    cbdPercent,
    "primaryImage": media[_type == "image"][0] {
      alt,
      "url": asset->url
    }
  }
`)

// All site users — for Phase 3 owner approval workflow in Studio
export const ALL_SITE_USERS_QUERY = defineQuery(`
  *[_type == "siteUser"] | order(_createdAt desc) {
    _id,
    email,
    name,
    status,
    _createdAt
  }
`)
```

Do NOT add `urlFor` calls here — this file is GROQ queries only. The `urlFor` helper from `src/sanity/lib/image.ts` is for frontend image transformations and is separate.
