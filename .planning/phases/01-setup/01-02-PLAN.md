<task type="auto">
  <name>Task 1: Generate NEXTAUTH_SECRET and create Sanity API token</name>
  <files>.env.local</files>
  <action>
    1. Generate a cryptographically strong secret for NEXTAUTH_SECRET:
    ```bash
    openssl rand -base64 32
    ```
    Copy the output — this is the NEXTAUTH_SECRET value.

    2. Open manage.sanity.io in a browser:
       - Navigate to the Pure Pressure project
       - Go to API > Tokens
       - Click "Add API token"
       - Name: `next-app-viewer`
       - Permissions: Viewer
       - Click "Save" and copy the token value (shown once only)

    3. Update `.env.local` with the real values:
    ```bash
    NEXT_PUBLIC_SANITY_PROJECT_ID=YOUR_PROJECT_ID_FROM_PLAN_01
    NEXT_PUBLIC_SANITY_DATASET=production
    SANITY_API_TOKEN=THE_TOKEN_FROM_STEP_2
    NEXTAUTH_SECRET=THE_SECRET_FROM_STEP_1
    NEXTAUTH_URL=http://localhost:3000
    ```
    Replace placeholders with actual values. NEXTAUTH_URL stays as localhost for now — it will be updated to the production domain after deployment.

    4. Configure Sanity CORS origins for localhost (run from project root):
    ```bash
    npx sanity@latest cors add http://localhost:3000 --credentials
    ```
    This whitelists localhost:3000 with credentials so the embedded Studio can authenticate locally.
  </action>
  <verify>
    ```bash
    # Verify dev server still works with populated env vars
    npm run dev
    ```
    - `http://localhost:3000` loads without errors
    - `http://localhost:3000/studio` loads Sanity Studio (should now be able to log in)
  </verify>
  <done>
    - `.env.local` has all 5 keys populated with real values (not blank or placeholder)
    - CORS origin `http://localhost:3000` appears in manage.sanity.io > API > CORS Origins with credentials enabled
    - Local dev server works with the populated env vars
  </done>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Connect GitHub repo to Vercel and deploy</name>
  <what-built>
    The project is code-complete and env vars are ready. This step connects the GitHub repository
    to Vercel for the first deployment. This requires browser-based UI interaction on Vercel.
  </what-built>
  <how-to-verify>
    Step 1 — Push current code to GitHub (if not already pushed):
    ```bash
    git add -A
    git commit -m "feat: scaffold Next.js 15 + Sanity embedded studio"
    git push origin main
    ```

    Step 2 — Go to https://vercel.com/new and import the GitHub repository.
    - Connect your GitHub account if needed
    - Select the `marcosweb` repository
    - Leave Framework Preset as "Next.js"
    - Do NOT add environment variables yet (we will do that in the next step)
    - Click "Deploy"
    - Wait for the deployment to complete
    - Note the assigned Vercel URL (e.g., `https://marcosweb.vercel.app` or `https://marcosweb-xyz.vercel.app`)

    Step 3 — Go to Vercel project > Settings > Environment Variables and add all 5 keys:
    | Key | Value |
    |-----|-------|
    | NEXT_PUBLIC_SANITY_PROJECT_ID | (from .env.local) |
    | NEXT_PUBLIC_SANITY_DATASET | production |
    | SANITY_API_TOKEN | (from .env.local) |
    | NEXTAUTH_SECRET | (from .env.local) |
    | NEXTAUTH_URL | https://YOUR-VERCEL-DOMAIN.vercel.app |

    For NEXTAUTH_URL: use the exact production domain Vercel assigned (no trailing slash).
    Scope all variables to "Production" (and optionally "Preview").

    Step 4 — Trigger a redeploy so the env vars take effect:
    - Vercel Dashboard > Deployments > click the latest deployment > "Redeploy"
    - OR push an empty commit: `git commit --allow-empty -m "chore: trigger redeploy with env vars" && git push`

    Step 5 — Add the production CORS origin to Sanity:
    ```bash
    npx sanity@latest cors add https://YOUR-VERCEL-DOMAIN.vercel.app --credentials
    ```
    Replace with your actual Vercel domain.

    Step 6 — Update .env.local to record the production URL:
    Change: `NEXTAUTH_URL=http://localhost:3000`
    To keep both: leave localhost for local dev, but document the prod URL in .env.example comment.

    Step 7 — Type "done" with your Vercel URL to continue.
  </how-to-verify>
  <resume-signal>Type "done: https://your-vercel-domain.vercel.app" so the URL is captured for the summary</resume-signal>
</task>
