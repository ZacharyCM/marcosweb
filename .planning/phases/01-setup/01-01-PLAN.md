<task type="auto">
  <name>Task 2: Configure project structure, embedded Studio, Sanity client, and env files</name>
  <files>

    src/app/(app)/page.tsx,

  <action>
    Perform the following file operations in order:

    1. CREATE directory `src/app/(app)/` and file `src/app/(app)/layout.tsx`:
    ```typescript
    import "@/styles/globals.css";

    export default function AppLayout({ children }: { children: React.ReactNode }) {
      return (
        <html lang="en">
          <body>{children}</body>
        </html>
      );
    }
    ```
    NOTE: This route group layout isolates Tailwind globals from the studio route. The studio must NOT inherit this layout — keep it outside the `(app)` group.

    2. If a root `src/app/layout.tsx` was generated by create-next-app, DELETE it (or move its content into the `(app)` layout). The root `app/` directory should have NO layout.tsx so the studio has a clean DOM.

    3. MOVE or RECREATE the home page at `src/app/(app)/page.tsx` (replacing any auto-generated `src/app/page.tsx`):
    ```typescript
    export default function Home() {
      return (
        <main className="flex min-h-screen items-center justify-center">
          <h1 className="text-2xl font-bold">Pure Pressure — Coming Soon</h1>
        </main>
      );
    }
    ```

    4. CREATE directory `src/app/studio/[[...index]]/` and file `src/app/studio/[[...index]]/page.tsx`:
    ```typescript
    "use client";

    import { NextStudio } from "next-sanity/studio";
    import config from "../../../../sanity.config";

    export { metadata, viewport } from "next-sanity/studio";

    export default function StudioPage() {
      return <NextStudio config={config} />;
    }
    ```
    CRITICAL: `"use client"` must be the first line. Studio is browser-only; rendering it as a Server Component causes build failure with `window is not defined`.

    5. VERIFY `src/styles/globals.css` contains `@import "tailwindcss"` as the first line (create-next-app with --tailwind should have done this). If it contains old `@tailwind base; @tailwind components; @tailwind utilities` directives instead, replace with `@import "tailwindcss";`.

    6. CREATE `sanity.config.ts` at the project root (replacing any auto-generated one from `sanity init` if it used `deskTool`):
    ```typescript
    import { defineConfig } from "sanity";
    import { structureTool } from "sanity/structure";

    export default defineConfig({
      name: "default",
      title: "Pure Pressure",
      projectId: process.env.NEXT_PUBLIC_SANITY_PROJECT_ID!,
      dataset: process.env.NEXT_PUBLIC_SANITY_DATASET || "production",
      basePath: "/studio",
      plugins: [structureTool()],
      schema: { types: [] },
    });
    ```
    Use `structureTool` from `sanity/structure` (NOT `deskTool` from `sanity/desk` — deprecated since v3.20.0). `basePath: "/studio"` MUST match the Next.js route exactly — a mismatch causes Studio internal nav to 404.

    7. CREATE `src/sanity/env.ts`:
    ```typescript
    export const apiVersion =
      process.env.NEXT_PUBLIC_SANITY_API_VERSION || "2024-07-11";
    export const dataset = process.env.NEXT_PUBLIC_SANITY_DATASET!;
    export const projectId = process.env.NEXT_PUBLIC_SANITY_PROJECT_ID!;
    ```

    8. CREATE directory `src/sanity/lib/` and file `src/sanity/lib/client.ts`:
    ```typescript
    import { createClient } from "next-sanity";
    import { apiVersion, dataset, projectId } from "../env";

    export const client = createClient({
      projectId,
      dataset,
      apiVersion,
      useCdn: true,
    });
    ```

    9. CREATE `.env.example` at project root (committed to git — no secrets):
    ```bash
    # Sanity CMS
    NEXT_PUBLIC_SANITY_PROJECT_ID=
    NEXT_PUBLIC_SANITY_DATASET=production
    SANITY_API_TOKEN=

    # Auth.js (NextAuth)
    NEXTAUTH_SECRET=
    NEXTAUTH_URL=
    ```

    10. VERIFY `.gitignore` contains `.env.local` (create-next-app adds this automatically — confirm it's there, do not remove it).

    11. UPDATE `next.config.ts` to add Sanity image CDN domain allowlist for future phases:
    Read the existing next.config.ts first, then add:
    ```typescript
    import type { NextConfig } from "next";

    const nextConfig: NextConfig = {
      images: {
        remotePatterns: [
          {
            protocol: "https",
            hostname: "cdn.sanity.io",
          },
        ],
      },
    };

    export default nextConfig;
    ```
    This prevents Next.js Image component errors in Phase 4 when rendering Sanity-hosted product images.

    12. CREATE `.env.local` at project root (will be gitignored) — populate with the Sanity project ID noted during `sanity init` in Task 1:
    ```bash
    NEXT_PUBLIC_SANITY_PROJECT_ID=REPLACE_WITH_YOUR_PROJECT_ID
    NEXT_PUBLIC_SANITY_DATASET=production
    SANITY_API_TOKEN=
    NEXTAUTH_SECRET=
    NEXTAUTH_URL=http://localhost:3000
    ```
    Replace `REPLACE_WITH_YOUR_PROJECT_ID` with the actual project ID from Task 1.
