---

**FLOW 1 — Age gate and registration (new user)**

Open an incognito window for this flow.

1. Navigate to the production URL (no cookies)
2. EXPECTED: Redirected to `/age-gate`. Page shows "I am 21 or older" button. Footer shows "For adults 21+ only. For use where legal."
3. Click "I am 21 or older"
4. EXPECTED: Redirected to `/login`. Check DevTools Application > Cookies: `age-verified` cookie should be set.
5. Click "Request an account" (or navigate to `/register`)
6. Fill in name, email (use a test email you control), and password
7. Submit the form
8. EXPECTED: Redirected to `/pending`. Page shows a "pending approval" message.
9. Sign out from the pending page
10. EXPECTED: Redirected to `/login` (not a loop back to `/pending`)

PASS if: age gate redirects, cookie sets, registration completes, /pending loads, sign-out works.

---

**FLOW 2 — Owner Sanity Studio (product creation and user approval)**

Use a normal (non-incognito) browser tab for this flow.

1. Navigate to `<production-url>/studio`
2. EXPECTED: Studio loads without CORS errors. No blank screen, no 403 errors in browser DevTools Network tab.
3. In Studio, navigate to the siteUser document for the test user registered in Flow 1
4. Change their status from "pending" to "approved" and click Publish
5. EXPECTED: Document saves and publishes without errors
6. In Studio, create a new product document:
   - Name: "Test Product" (or any name)
   - Strain type: sativa (or any)
   - Price: 25.00
   - THC%: 20
   - CBD%: 1
   - Description: any text
   - Effects: add at least one (e.g., "Relaxing")
   - Upload a primary image
7. Click Publish
8. EXPECTED: Product saves and publishes without errors. Image upload completes.

PASS if: Studio loads, siteUser approval publishes, product with image publishes — all without errors.

---

**FLOW 3 — Approved user login and menu browsing**

Use the incognito window from Flow 1 (the user you registered and approved).

1. Navigate to the production URL (age cookie already set from Flow 1)
2. EXPECTED: Lands on `/login`
3. Enter the approved user's email and password
4. Submit the login form
5. EXPECTED: Redirected to `/menu`. No "Configuration" error. No spinner that freezes.
6. Verify the menu shows at least one strain carousel with the test product created in Flow 2
7. Each product card should show: image, product name, strain badge, price, THC%, CBD%
8. Click the test product card
9. EXPECTED: Modal opens with dimmed backdrop. Shows product name, description, effects chips, and the uploaded image in the media gallery.
10. Press Escape
11. EXPECTED: Modal closes. Menu still visible.
12. Sign out (look for sign-out button or navigate to `/api/auth/signout`)
13. EXPECTED: Redirected to `/login`

PASS if: login redirects to /menu, carousels visible with products and images, modal opens with content, escape closes modal, sign-out works.

---

**FLOW 4 — Denied user**

For this flow, you need a second test account OR you can temporarily change the Flow 1 user to "denied" in Studio, test, then re-approve.

1. In Sanity Studio, set a user's status to "denied" and publish
2. Navigate to the production URL login page
3. Enter the denied user's credentials
4. EXPECTED: Login page shows "Your access was not approved" message (or equivalent denied error message)
5. EXPECTED: No redirect loop — page stays on `/login` with the error message
6. (Optional) Navigate to `/menu` directly while denied
7. EXPECTED: Redirected to `/login`

PASS if: denied user sees error message, no redirect loop.

---

**FLOW 5 — Direct URL access protection**

In an incognito window (no session, but age-verified cookie is fine):

1. Navigate directly to `<production-url>/menu`
2. EXPECTED: Redirected to `/login` (not a 500, not the menu)
3. Log in with a valid approved account
4. While logged in, navigate to `<production-url>/age-gate`
5. EXPECTED: Age gate page loads (PUBLIC_PATHS allows it)
6. Navigate to `<production-url>/privacy`
7. EXPECTED: Privacy policy page loads without requiring login

PASS if: /menu redirects unauthenticated users, /age-gate and /privacy are accessible without login.
