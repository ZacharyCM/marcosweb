6. If it already exists with a value: confirm the value matches `.env.local` NEXTAUTH_SECRET and leave it.

---

**STEP 2 — Vercel: Add/confirm AUTH_URL**

1. In the same Vercel Environment Variables panel
2. Check if `AUTH_URL` exists
3. If it does NOT exist: Add `AUTH_URL` with value `https://<your-production-domain>.vercel.app` (use the actual production URL, not localhost). Set environment: Production.
4. If it already exists: confirm it points to the production Vercel domain (not localhost, not a preview URL).

This is the critical fix for the signIn "Configuration" error on Next.js 16 (issue #13388). AUTH_URL must be the exact production domain including `https://`.

---

**STEP 3 — Vercel: Confirm SANITY_WRITE_TOKEN**

1. In Vercel Environment Variables: check if `SANITY_WRITE_TOKEN` exists (distinct from SANITY_API_TOKEN)
2. If it does NOT exist: Add `SANITY_WRITE_TOKEN` with the value from `.env.local` (the `skiw8t...` token). Set environment: Production.

---

**STEP 4 — Sanity: Verify CORS origins include production domain with credentials**

1. Open https://sanity.io/manage
2. Select the Pure Pressure project (project ID: vgi4fxay)
3. Go to API > CORS Origins
4. Look for the production Vercel domain (e.g., `https://pure-pressure.vercel.app` or similar)
5. If the production domain is NOT in the list: run this command to add it (replace the URL with your actual Vercel domain):
   ```bash
   npx sanity cors add https://<your-production-domain>.vercel.app --credentials
   ```
   Or add it manually via the dashboard UI with "Allow credentials" checked.
6. Confirm `http://localhost:3000` is also in the list with credentials (for local dev).

IMPORTANT: The domain must have "Allow credentials" checked. Adding without credentials will still cause Studio to fail.

**STEP 5 — Sanity: Verify write token role**
