<task type="auto">
  <name>Task 2: Create sanity-write.ts client and auth-types.ts augmentation</name>
  <files>src/lib/sanity-write.ts, src/lib/auth-types.ts, .env.example</files>
  <action>
    1. Create directory src/lib/ if it does not exist.

    2. Create src/lib/sanity-write.ts:
    ```typescript
    import "server-only"
    import { createClient } from "next-sanity"
    import { apiVersion, dataset, projectId } from "@/sanity/env"

    export const sanityWriteClient = createClient({
      projectId,
      dataset,
      apiVersion,
      useCdn: false, // Always fetch fresh data for auth operations — never stale CDN cache
      token: process.env.SANITY_WRITE_TOKEN, // Editor role token — server-only, NOT NEXT_PUBLIC_
    })
    ```
    The `import "server-only"` line prevents this module from being imported in client components. This is a hard guard — if any client component tries to import sanityWriteClient, the build will fail with a clear error.

    3. Create src/lib/auth-types.ts:
    ```typescript
    import type { DefaultSession } from "next-auth"

    declare module "next-auth" {
      interface Session {
        user: {
          id: string
          status: string
        } & DefaultSession["user"]
      }
      interface User {
        status: string
      }
    }

    declare module "next-auth/jwt" {
      interface JWT {
        id: string
        status: string
      }
    }
    ```
    This module augmentation extends Auth.js built-in types so that `session.user.id` and `session.user.status` are recognized by TypeScript without casting. No import is needed elsewhere — TypeScript picks up declare module augmentations automatically when the file is included in the tsconfig.

    4. Update .env.example — add the new required variables:
    ```
    # Sanity CMS
    NEXT_PUBLIC_SANITY_PROJECT_ID=
    NEXT_PUBLIC_SANITY_DATASET=production
    SANITY_API_TOKEN=
    SANITY_WRITE_TOKEN=

    # Auth.js (NextAuth v5)
    AUTH_SECRET=
    AUTH_URL=
    ```
    Remove NEXTAUTH_SECRET and NEXTAUTH_URL (Auth.js v5 uses AUTH_SECRET and AUTH_URL). Keep the Sanity read token (SANITY_API_TOKEN) alongside the new write token (SANITY_WRITE_TOKEN).

    5. Verify TypeScript compiles cleanly after the type augmentation:
    ```bash
    npx tsc --noEmit
    ```
